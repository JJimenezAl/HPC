Description Josh Bowden 2016-09-11 21:06:15 MDT 
Hi

I'm trying to determine how much memory a job has been given.

I request memory like this
#SBATCH --mem=64GB

And the way I can find out is using (with some extra processing):
MEMREQUESTED=`sacct -o reqmem -j $SLURM_JOB_ID`

However, sacct seems to give bogus results on occasion. e.g. not 64Gn, but perhaps 4 or nothing. 

Is there a better way to determine memory allocated to a job? 

My version of SLURM is:
 slurm 15.08.6

Thanks for your help with this.

Comment 3 Josh Bowden 2016-09-13 02:07:13 MDT 
Hi Alejandro,

I have found a workaround – put a “sleep 5” before MEMREQUESTED=`sacct -o reqmem -n -j $SLURM_JOB_ID`

Our system is a bit slow to register its accounting db it seems.

I got a 7/7 pass test pass rate with the sleep pause. That is better than the 1/7 without it.

Cheers,
Josh

FYI:

And this is an example sbatch script that will show the problem on our system:

#!/bin/bash
#SBATCH --job-name="test_acctl"
#SBATCH --time=00:05:00
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=2
#SBATCH --mem=8GB
#SBATCH --output=test3_acctl_%j.out

# set the temporary file location
echo "TMPDIR = $TMPDIR"
echo "WORKDIR = $WORKDIR"
echo "hostname = " ; hostname

# go to the job submission directory; you can change this to your project directory
cd $SLURM_SUBMIT_DIR
module load R
module load jdk

LAUNCHTHREADS=$SLURM_NTASKS_PER_NODE
MEMFORR=4
MEMREQUESTED=`sacct -o reqmem -n -j $SLURM_JOB_ID`
echo "MEMREQUESTED = $MEMREQUESTED"
MEMREQUESTED=`echo ${MEMREQUESTED} | grep G | sed -e 's/Gn//'`
echo "MEMREQUESTED = $MEMREQUESTED"
MEMREQUESTED=$(( $MEMREQUESTED-$MEMFORR ))
echo "MEMREQUESTED = $MEMREQUESTED"

Comment 4 Alejandro Sanchez 2016-09-13 05:51:57 MDT 
Josh yes... it's just a matter of time. `sacct` command retrieves the ReqMem value from the <cluster>_job_table record for the submitted job record, and the record is created when jobacct_storage_p_job_start() runs. But while it runs the job script is run in parallel so if you need to `sacct` inside the job script itself you've to manually add some delay or wait for the job script to finish to `sacct` this job record. Please, let me know if this makes sense to you. Thanks.

Comment 6 Alejandro Sanchez 2016-09-13 07:02:38 MDT 
--mem is also available from SLURM_MEM_PER_NODE env variable. So you could just set MEMREQUESTED = $SLURM_MEM_PER_NODE.
Comment 7 Alejandro Sanchez 2016-09-13 07:31:45 MDT 
Requested job memory is also available through the job submit plugins. For instance with Lua you can access the job_desc.pn_min_memory field.
